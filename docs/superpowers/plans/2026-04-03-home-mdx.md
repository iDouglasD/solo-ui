# Home.mdx Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a terminal-themed `Home.mdx` page in Storybook as the entry point for developers consuming solo-ui.

**Architecture:** A single unattached MDX file at `packages/docs/src/Home.mdx`. Layout built entirely with JSX `<div>` elements using Tailwind utility classes and `@solo-ui/tokens` color tokens. No new components, no logic — pure static content.

**Tech Stack:** Storybook 10 MDX, Tailwind CSS v4, `@storybook/addon-docs/blocks`

---

## File Map

| Action | Path | Responsibility |
|--------|------|----------------|
| Create | `packages/docs/src/Home.mdx` | Entire Home page — hero, install, components |

No other files need to be modified. The existing Storybook glob `../src/**/*.@(stories.@(ts|tsx)|mdx)` already picks up any `.mdx` file under `src/`.

---
